### 基于工具学习的金融虚假信息检测：Context-Aware Tool Orchestrator 框架

#### （1）背景  
金融虚假信息检测面临语义隐晦性、知识动态性与修辞操纵性三重挑战。传统端到端分类模型难以捕捉金融文本中隐含的上下文依赖（如市场事件关联）、跨时间逻辑矛盾（如业绩承诺与历史数据冲突）及修辞策略（如情感煽动与事实模糊化）。现有RAG方法仅提供静态检索增强，缺乏对分析路径的动态规划能力；而工具调用范式多采用固定流水线，无法根据文本特性自适应选择验证视角。本文提出**上下文感知的工具编排器（Context-Aware Tool Orchestrator, CATO）**，通过元认知规划驱动的工具动态组合机制，实现多粒度、可解释的虚假信息检测。

#### （2）动机  
**核心挑战**：  
① 金融虚假信息常依赖**缺失的上下文知识**进行隐蔽传播（如孤立陈述“某公司获政府巨额补贴”需结合政策文件验证）；  
② 虚假模式具有**高度异质性**（同一文本可能同时涉及事实扭曲、时间逻辑矛盾与修辞操纵），需动态组合分析视角而非固定工具序列。  

**解决思路**：  
① 构建**检索增强的元认知规划器**，将外部知识库作为“认知扩展”，引导LLM生成上下文感知的工具调用序列；  
② 设计**证据传播有向无环图（Evidence Propagation DAG）**，使工具间形成逻辑依赖与置信度加权融合，避免工具孤岛效应。

#### （3）模块设计  

##### 3.1 上下文感知检索模块（Context-Aware Retrieval Module）  
**挑战**：金融文本的虚假性常隐含于跨文档关联中（如孤立推文“央行将降息”需结合货币政策会议纪要验证），传统向量检索难以捕获此类隐式逻辑关联。  
**思路**：采用**双通道语义-逻辑联合检索**，既计算文本表征相似度，又通过规则模板提取实体-事件三元组进行结构化匹配，确保检索结果覆盖事实依据与逻辑上下文。  
**实现**：输入待检测文本 $T$，首先通过文本表征模型（如bge-large）检索知识库中语义相似的Top-K样本；同时解析 $T$ 中的金融实体（公司/政策/指标）与事件类型（并购/财报/监管），匹配知识库中具有相同实体-事件组合的样本。最终融合两通道结果，返回包含标签、来源可信度及时间戳的增强上下文集 $\mathcal{C} = \{ (c_i, y_i, \text{cred}_i, t_i) \}_{i=1}^K$。

##### 3.2 元认知规划器（Meta-Cognitive Planner）  
**挑战**：LLM难以自主判断“何时调用何种工具”——例如对“季度利润增长200%"需优先验证事实一致性，而对“内幕消息：股价将暴涨”则需侧重来源可信度分析。  
**思路**：将检索上下文 $\mathcal{C}$ 作为“认知锚点”，引导LLM进行**反事实推理式规划**：生成“若该文本为真，则应满足哪些条件”的假设链，并映射至工具调用序列。  
**实现**：构造提示词模板：  
> “基于以下上下文 $\mathcal{C}$，分析文本 $T$ 的潜在虚假风险。请按逻辑顺序列出需验证的维度（如事实一致性、时间逻辑等），并说明每个维度的验证依据（引用 $\mathcal{C}$ 中的具体证据）。输出JSON格式的工具调用计划：{‘tools’: [‘tool_name1’, ‘tool_name2’...], ‘dependencies’: [(‘tool1’, ‘tool2’)]}”  
LLM输出结构化计划，显式定义工具间的DAG依赖关系（如“事实验证”需先于“矛盾传播”）。

##### 3.3 证据传播工具集（Evidence Propagation Toolset）  
**挑战**：单一工具视角易产生误判（如语义连贯的文本可能事实错误），需设计工具间证据传递机制以实现交叉验证。  
**思路**：构建10类专业化工具，按**验证粒度**组织为四级DAG（见表1），每级工具输出结构化证据（置信度+关键片段+矛盾点），并通过**置信度加权传播**机制向下游工具传递。  

| 层级 | 工具名称 | 功能描述 | 证据输出格式 |
|------|----------|----------|--------------|
| L1（基础锚定） | **Contextual Grounding Tool (CGT)** | 将 $T$ 与 $\mathcal{C}$ 对齐，识别缺失上下文 | `{missing_context: [实体/事件], anchor_evidence: str}` |
|  | **Source Credibility Propagator (SCP)** | 基于 $\mathcal{C}$ 中来源可信度加权传播至 $T$ | `{cred_score: float, propagation_path: list}` |
| L2（语义-语用解耦） | **Semantic Coherence Analyzer (SCA)** | 检测内部逻辑矛盾（如“亏损”与“创收新高”共现） | `{contradiction_pairs: [(span1, span2)], coherence_score: float}` |
|  | **Pragmatic Intent Decoder (PID)** | 识别修辞操纵（情感煽动/模糊化表述） | `{manipulation_type: str, intensity: float}` |
| L3（事实-时间验证） | **Factual Consistency Verifier (FCV)** | 对比 $T$ 与 $\mathcal{C}$ 中事实陈述 | `{verified_facts: list, conflicts: [(claim, evidence)]}` |
|  | **Temporal Logic Validator (TLV)** | 验证时间序列合理性（如“连续三季增长”需匹配财报时间线） | `{temporal_chain: list, violation_points: list}` |
|  | **Rhetorical Manipulation Detector (RMD)** | 检测特定金融修辞陷阱（如“历史性机遇”“内部消息”） | `{trap_keywords: list, risk_level: str}` |
| L4（高阶推理） | **Expectation Violation Analyzer (EVA)** | 识别违背市场常识的陈述（如“零风险高收益”） | `{violation_type: str, norm_reference: str}` |
|  | **Contradiction Propagation Engine (CPE)** | 融合L1-L3证据，传播矛盾至全局 | `{global_contradiction_graph: dict, propagation_weight: float}` |

*表1：工具DAG结构（L1→L2→L3→L4为有向边，同级工具并行执行）*

##### 3.4 工具路由器（Tool Router with DAG Scheduler）  
**挑战**：元认知规划器输出的工具序列需满足DAG约束（如FCV必须在CGT之后执行），且需处理工具执行失败的回退机制。  
**思路**：设计**拓扑排序调度器**，将规划器输出的依赖关系转换为执行队列，并引入**证据阈值触发机制**——若上游工具置信度低于阈值 $\tau$，则激活备用工具路径。  
**实现**：解析规划器JSON输出，构建DAG拓扑序；按序调用工具，实时监控输出证据质量。例如：若SCP输出的可信度 $<0.3$，则跳过L2工具直接激活L4的EVA进行常识验证。

##### 3.5 多视角证据融合器（Multi-Perspective Evidence Fusion）  （此处你可以先实现一个简单的版本，直接复用你之前的 prompt 预测，就是把前面各个模块的输出加到你的 prompt 即可，这样可以尽量保留之前的效果，这个模块不是重点）
**挑战**：不同工具证据存在量纲差异（如置信度0-1与风险等级高/中/低），需统一表征进行决策。  
**思路**：采用**证据理论（Dempster-Shafer Theory）** 将各工具输出映射为基本概率分配（BPA），通过Dempster合成规则融合多源证据，输出最终虚假概率。  
**实现**：将每个工具证据转换为BPA：$m_{\text{fake}} = \text{tool\_confidence} \times \text{weight}_{\text{tool}}$，其中权重由工具在DAG中的层级与历史准确率动态调整。合成后若 $m_{\text{fake}} > 0.65$ 则判为虚假。

#### （4）核心切入点与样例  
**切入点1：上下文缺失导致的误判**  
- **问题**：孤立文本“XX科技获国家专项补贴50亿元”看似合理，但缺失政策背景（如该公司正被审计）将导致漏检。  
- **CATO解决**：CGT工具检索到同期“财政部暂停对该行业补贴”的政策文件，SCP工具识别来源为匿名推特（可信度0.2），触发TLV验证时间矛盾（政策暂停日早于声明日），最终CPE传播矛盾至全局证据。  

**切入点2：分析角度的自适应缺失**  
- **问题**：文本“内幕消息：明日涨停”与“财报显示净利润增长200%"需不同验证路径，固定工具序列无法适配。  
- **CATO解决**：元认知规划器基于检索结果（前者匹配到“内幕交易警示”样本，后者匹配到财报数据库）生成差异化DAG：前者优先调用SCP→RMD→EVA，后者调用CGT→FCV→TLV，实现角度自适应。

#### （5）预期实验  
在金融推特/新闻数据集上对比三类基线：① 纯LLM分类器（Zero-shot）；② 静态RAG增强分类；③ 固定工具流水线（找点multi-agent的文章）。评估指标包括：  
- **主指标**：F1-score（虚假类）、Accuracy等；  
- **可解释性指标**：工具调用路径与人工标注的虚假原因匹配度（通过专家评估）；  
- **消融实验**：移除元认知规划器、禁用证据传播机制，验证各模块贡献。  
预期CATO在F1-score上提升x-x%，且在“上下文依赖型虚假信息”子集上优势显著（+x%），证明工具动态编排对复杂虚假模式的捕获能力。  
- **LLM底座鲁棒性**：不同的LLM底座对模型效果的影响。
- **参数敏感性分析**：不同的超参数对模型效果的影响；
