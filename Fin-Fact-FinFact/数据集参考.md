## 📝 数据样本展示

### 样本 1: False 标签示例

**声明 (Claim)**:
> "在奥巴马任期内，美国国债增加了 1 万亿美元。"

**标签**: `false`

**科学摘要 (Sci Digest)**:
> 实际上，在奥巴马总统任期（2009-2017）内，美国国债增加了约 9 万亿美元，而非 1 万亿。

**证据 (Evidence)**:
1. 根据美国财政部数据，2009年1月国债总额为10.6万亿美元
2. 2017年1月国债总额为19.9万亿美元
3. 增加额约为9.3万亿美元

**主题 (Issues)**: `['National', 'debt', 'economy']`

**图片**: 包含国债增长趋势图表

---

### 样本 2: True 标签示例

**声明 (Claim)**:
> "苹果公司是美国最有价值的上市公司之一。"

**标签**: `true`

**科学摘要 (Sci Digest)**:
> 根据市值数据，苹果公司确实是美国乃至全球市值最高的上市公司之一。

**证据 (Evidence)**:
1. 2023年苹果市值超过2.5万亿美元
2. 长期位居标普500指数市值榜前列
3. 财报显示持续盈利能力

**主题 (Issues)**: `['share', 'profit', 'investment']`

**图片**: 苹果公司股价走势图

---

### 样本 3: NEI 标签示例

**声明 (Claim)**:
> "某小型银行计划在下个季度调整利率政策。"

**标签**: `NEI` (Not Enough Information)

**科学摘要 (Sci Digest)**:

**证据 (Evidence)**:
- 未找到官方声明
- 缺乏权威媒体报道
- 相关金融监管文件未提及

**主题 (Issues)**: `['interest', 'bank', 'credit']`

**图片**: 无

## 📐 数据字段详解

### 核心字段

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `url` | String | ✅ | 核查文章原始链接 | https://www.snopes.com/... |
| `claim` | String | ✅ | 待验证的声明文本 | "公司X的利润增长了50%" |
| `label` | String | ✅ | 真实性标签 | "true" / "false" / "NEI" |
| `author` | String | ✅ | 声明作者或来源 | "Social Media" / "News" |
| `posted` | String | ✅ | 声明发布日期 | "2023-05-15" |

### 核查字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `sci_digest` | String | ✅ | 科学摘要，简要说明核查结果 |
| `justification` | String | ✅ | 详细解释文本，逻辑推理过程 |
| `evidence` | Array | ✅ | 证据列表，每项包含 sentence 和 url |

### 扩展字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `issues` | Array | ✅ | 主题标签列表 |
| `image_data` | Array | ❌ | 图片数据（75.1%记录包含） |

### Evidence 结构

```json
"evidence": [
  {
    "sentence": "根据美国劳工统计局的数据...",
    "url": "https://www.bls.gov/..."
  }
]
```

**特点**：
- 平均每条记录 3-5 条证据
- 证据来源多样：政府网站、学术机构、权威媒体
- 支持证据溯源和信息追踪

### Image Data 结构

```json
"image_data": [
  {
    "image_url": "https://example.com/image.jpg",
    "image_caption": "图表显示2020-2023年股价走势"
  }
]
```

**特点**：
- 图片类型：社交媒体截图、数据可视化、新闻配图
- 与声明强相关（非装饰性图片）
- 支持多模态建模

- ✅ **专业标注**：来自 Snopes/PolitiFact 专业团队

## 💻 快速统计代码

### 基础统计

```python
import json
from collections import Counter

# 加载数据
with open('finfact.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 基础统计
print(f"总记录数: {len(data)}")
print(f"标签分布: {Counter(d['label'] for d in data)}")
print(f"包含图片: {sum(1 for d in data if d.get('image_data'))} ({sum(1 for d in data if d.get('image_data'))/len(data)*100:.1f}%)")

# 主题统计
issues = []
for d in data:
    issues.extend(d.get('issues', []))
print(f"\nTop 10 主题:")
for issue, count in Counter(issues).most_common(10):
    print(f"  {issue}: {count}")
```

### 文本长度分析

```python
import numpy as np

# Claim 长度
claim_lengths = [len(d['claim'].split()) for d in data]
print(f"Claim 词数统计:")
print(f"  平均: {np.mean(claim_lengths):.1f}")
print(f"  中位数: {np.median(claim_lengths):.1f}")
print(f"  最小: {np.min(claim_lengths)}")
print(f"  最大: {np.max(claim_lengths)}")

# Evidence 数量
evidence_counts = [len(d.get('evidence', [])) for d in data]
print(f"\nEvidence 数量统计:")
print(f"  平均: {np.mean(evidence_counts):.1f}")
print(f"  中位数: {np.median(evidence_counts):.1f}")
```

### 数据切分

```python
from sklearn.model_selection import train_test_split

# 按标签分层切分（80/10/10）
train_data, temp_data = train_test_split(data, test_size=0.2, stratify=[d['label'] for d in data], random_state=42)
val_data, test_data = train_test_split(temp_data, test_size=0.5, stratify=[d['label'] for d in temp_data], random_state=42)

print(f"训练集: {len(train_data)}")
print(f"验证集: {len(val_data)}")
print(f"测试集: {len(test_data)}")
```

## 🎯 使用技巧

### 1. 数据预处理建议

```python
# 清洗文本
def clean_text(text):
    # 移除多余空格
    text = ' '.join(text.split())
    # 统一标点
    text = text.replace('"', '"').replace('"', '"')
    return text

# 处理图片缺失
def has_valid_image(item):
    return bool(item.get('image_data') and len(item['image_data']) > 0)
```

### 2. 特征工程

- **文本特征**：使用 BERT/RoBERTa 提取语义特征
- **图片特征**：使用 CLIP/ResNet 提取视觉特征
- **证据特征**：计算证据相关度得分
- **主题特征**：one-hot 编码或嵌入主题标签

### 3. 模型选择

| 任务类型 | 推荐模型 | 输入 |
|----------|----------|------|
| 纯文本分类 | BERT, RoBERTa, DeBERTa | claim + evidence |
| 多模态融合 | CLIP, VisualBERT, LXMERT | claim + image |
| 生成式核查 | GPT-4, Gemini, LLaMA | claim + evidence + image |
| 证据检索 | DPR, ColBERT | claim → evidence |
