## 基于多智能体检索增强框架的健康虚假信息反击生成

##  Multi-Agent Retrieval-Augmented Framework for Evidence-Based Counterspeech Against Health Misinformation



该论文主要解决了LLM在健康医疗虚假信息时的三个核心痛点：

1. **证据来源的局限性**：
   - 单纯依靠LLM内部知识会产生幻觉。
   - 传统的**静态RAG**（查数据库）缺乏时效性（无法应对突发疫情新闻）。
   - 单纯的**动态RAG**（查搜索引擎）可能引入低质量或未经证实的来源。
   - **解决方案**：结合静态权威指南（如CDC、WHO文档）和动态实时搜索（DuckDuckGo API）。
2. **信息过载与冲突**：
   - 从多个来源检索到的信息可能冗余、杂乱甚至相互矛盾，直接喂给LLM会导致生成效果差。
   - **解决方案**：引入专门的“总结智能体”来清洗和融合证据。
3. **回复质量与可接受度**：
   - 生成的反击言论如果语气生硬、充满说教，用户很难接受。
   - **解决方案**：引入专门的“润色智能体”优化语气和结构。

![PixPin_2026-01-21_20-21-34](./pics/PixPin_2026-01-21_20-21-34.png)

#### 具体Agent分工：``输入帖子`` -> `检索(静态+动态) `->` 总结` ->`` 生成 ``->`` 润色`` -> 输出

1. **静态检索智能体 (Static Retrieval Agent)**
   - **职责**：在本地构建的权威医疗知识库（来自NIH, WHO等）中检索证据。
   - **技术**：混合检索（Hybrid Search），结合了向量相似度搜索（Embedding）和关键词匹配。
2. **动态检索智能体 (Dynamic Retrieval Agent)**
   - **职责**：获取实时互联网信息，应对最新的谣言或变种。
   - **技术**：调用DuckDuckGo Search API，并进行事实核查过滤（过滤掉低可信度来源）。
3. **总结智能体 (Summarization Agent)**
   - **职责**：解决“上下文过长”和“信息冗余”问题。
   - **动作**：接收前两个Agent检索到的Top-K证据，利用LLM进行去重、去除无关信息，生成一份精简的“最终证据摘要”。
4. **反击生成智能体 (Counter speech Generation Agent)**
   - **职责**：起草回复。
   - **Prompt策略**：使用结构化提示词，要求基于“最终证据摘要”直接反驳错误信息，并提供具体例子。
5. **润色智能体 (Refinement Agent)**
   - **职责**：优化回复的“人性化”程度。
   - **核心指标**：礼貌性（Politeness）、同理心、清晰度。
   - **动作**：检查初稿是否过于对抗性，将其重写为更易于被人类接受的语气，同时保留核心事实证据。





## Ask, Acquire, Understand: A Multimodal Agent-based Framework for Social Abuse Detection in Memes (Ask, Acquire, Understand：用于模因社会滥用检测的多模态多智能体框架)

该论文主要解决了多模态模因（Meme）检测中的以下核心痛点：

1.  **图像描述信息量不足**：
    *   现有的方法通常使用图像描述模型将图片转为文本，但生成的描述往往过于通用，缺乏检测社会滥用（如仇恨言论、讽刺）所需的关键细节和上下文。
    *   **解决方案**：提出“Ask, Acquire, Understand”框架，利用LLM作为智能体主动提出有见地的问题，引导视觉模型（LMM）关注并提取更深层的视觉细节。
2.  **现有模型缺乏泛化性与鲁棒性**：
    *   以往方法多在有限数据集上测试，且多模态融合模型难以处理文本与图像含义差异巨大的情况（如讽刺）。
    *   **解决方案**：构建基于零样本（Zero-shot）学习的多智能体系统，通过角色扮演和迭代对话，无需训练即可在不同类型的滥用检测任务中保持高性能。

![PixPin_2026-01-21_20-48-12](./pics/PixPin_2026-01-21_20-48-12.png)



#### 工作流：输入模因（图+文） -> 初始描述生成 -> 多智能体讨论与提问（Ask/Acquire循环） -> 总结与分类（Understand） -> 输出结果

1.  **视觉专家智能体 (Vision Expert)**
    *   **职责**：充当系统的“眼睛”，负责理解图像内容并提供视觉信息。
    *   **技术/动作**：使用大模型（如LLaVA-1.6-13B），首先生成图像的初始描述；随后在多轮对话中，接收其他智能体提出的具体问题（`<question>`标签），并仅针对问题进行回答，提供补充视觉细节。

2.  **多智能体对话模块 (Multi-Agent: Internet User & Internet Supervisor)**
    *   **职责**：模拟不同视角对模因进行分析和辩论，挖掘潜在的滥用含义。
    *   **技术/动作**：利用GPT-3.5进行角色扮演。
        *   **Internet User（互联网用户）**：从普通网民角度看模因。
        *   **Internet Supervisor（互联网监管者）**：从审核监管角度看模因。
        *   两者基于初始信息进行讨论，若信息不足，会生成针对图像的特定问题发送给“视觉专家”，获取新信息后继续下一轮讨论。

3.  **总结助手智能体 (Summary Assistant)**
    *   **职责**：整合所有信息并进行最终的分类判断。
    *   **技术/动作**：在对话结束后，利用GPT-3.5接收完整的对话历史（QA History）和模因基本信息，结合思维链（CoT）推理，根据特定任务定义（如是否包含仇恨、讽刺等）输出最终的分类结果（Yes/No）。











## ————看到这里————T2Agent: A Tool-augmented Multimodal Misinformation Detection Agent with Monte Carlo Tree Search (基于蒙特卡洛树搜索的工具增强型多模态虚假信息检测智能体)

该论文解决了多模态虚假信息检测中的核心痛点：
1. **现有方法的僵化与局限性**：
   - 现有的基于LLM的方法通常依赖固定的验证流程（如先查图后查文）或有限的工具集，缺乏处理复杂、多样化伪造类型（如文本错误、图片篡改、图文不符）的灵活性。
   - **解决方案**：引入基于蒙特卡洛树搜索（MCTS）的动态推理框架，根据当前证据动态规划验证路径，不再拘泥于固定顺序。
2. **混合伪造源的复杂性**：
   - 现实中的虚假信息往往由多种伪造手段混合而成，单一视角的检测难以奏效，且需要在“广泛探索不同伪造源”与“深入挖掘现有证据”之间取得平衡。
   - **解决方案**：提出“多源验证MCTS”，**将检测任务分解为针对不同伪造源（文本、视觉、跨模态）的子任务**，利用双重奖励机制（推理轨迹评分+结果置信度评分）平衡探索与利用。
3. **工具冗余带来的低效**：
   - 同时使用所有可用工具会导致上下文过长、效率降低且干扰模型判断。
   - **解决方案**：设计基于贪婪搜索的工具选择机制，针对特定数据集自动筛选最有效的工具子集。

#### 工作流：输入(多模态新闻) -> 初始评估(任务分解) -> MCTS动态验证循环(选择-扩展-模拟-反向传播) -> 决策融合 -> 输出(真/假)

1. **推理与规划智能体 (Reasoning Controller / LVLM)**
   - **职责**：作为核心大脑，负责在MCTS的每个节点中生成“思考-行动”对，并在模拟阶段执行具体验证逻辑。
   - **技术/动作**：利用大视觉语言模型（如GPT-4o, LLaVA），根据当前状态生成验证计划（Plan），调用工具（Action），并根据工具返回的观察结果（Observation）更新状态记忆。

2. **多源验证MCTS引擎 (Multi-source Verification MCTS Engine)**
   - **职责**：管理整个推理过程的宏观路径，决定下一步验证哪个潜在的伪造源（文本、图片或一致性）。
   - **技术/动作**：
     - **选择 (Selection)**：使用改进的UCT算法选择最有潜力的子任务节点。
     - **扩展与模拟 (Expansion & Simulation)**：调用推理智能体执行多步验证，直到得出结论或达到深度限制。
     - **剪枝策略**：一旦某个子任务以高置信度判定为“假”，立即终止其他分支的搜索（模拟人类专家的“一票否决”逻辑）。
     - **反向传播 (Backpropagation)**：利用双重奖励函数更新父节点的价值。

3. **评估与反馈模块 (Evaluation Module)**
   - **职责**：量化推理过程的质量和最终结论的可靠性，引导搜索方向。
   - **技术/动作**：
     - **推理轨迹评分 (Trajectory Score)**：评估推理路径的逻辑连贯性。
     - **结果置信度评分 (Confidence Score)**：评估证据对结论的支撑程度。

4. **扩展工具库 (Extensible Toolkit)**
   - **职责**：提供具体的证据收集和分析能力，作为智能体的“手”。
   - **技术/动作**：包含标准化接口的模块化工具，如网络搜索（Google/Wiki）、伪造检测（PSCC-Net）、反事实检测（LLaVA-34B）、图像理解、实体识别等。通过贪婪搜索预先筛选最佳工具组合。











## Toward Verifiable Misinformation Detection: A Multi-Tool LLM Agent Framework (走向可验证的虚假信息检测：一种多工具 LLM 智能体框架)

该论文主要解决了LLM在虚假信息检测中的两个核心痛点：

1.  **缺乏可验证性与透明度**：
    *   传统的检测方法通常只是输出一个简单的真/假标签，如同黑盒，缺乏中间推理过程和证据支持，用户难以信任。
    *   **解决方案**：设计一个基于ReAct框架的智能体，不仅给出结论，还提供完整的、可追溯的推理链条和证据日志（Evidence Logs），使检测过程透明化、可审计。
2.  **面对复杂/改写信息的脆弱性**：
    *   LLM容易产生幻觉，且受限于训练数据的时效性，难以应对最新或经过精心改写（Rewritten/Laundered）的虚假信息。
    *   **解决方案**：引入多工具协作机制，特别是实时网络搜索和数值验证工具，让智能体能够动态获取最新信息并进行精确的数值核查，从而在面对改写内容时保持鲁棒性。

![PixPin_2026-01-21_20-58-03](./pics/PixPin_2026-01-21_20-58-03.png)

#### 工作流：输入声明 -> 规划与分析 -> 多工具执行与证据收集 -> 综合与报告生成

1.  **规划与分析模块 (Planning and Analysis)**
    *   **职责**：作为“大脑”，负责解析输入声明，确定核心事实和数值点。
    *   **技术/动作**：分析输入内容，制定详细的验证计划，决定需要验证哪些具体的子声明（Sub-claims），以及针对每个子声明应该调用什么工具、按什么顺序调用。

2.  **多工具执行与证据收集模块 (Multi-Tool Execution and Evidence Gathering)**
    *   **职责**：作为“手脚”，执行验证计划并动态调整策略。
    *   **技术/动作**：基于ReAct框架，循环执行“观察-思考-行动”。
        *   **Web Search Tool (网络搜索工具)**：调用Tavily API进行实时搜索，提取关键文本和元数据，突破LLM知识截止限制。
        *   **Source Credibility Assessment Tool (来源可信度评估工具)**：将搜索到的来源与预定义的可信度数据库（包含2847个源）进行比对，将其分为高、中、低三级，赋予不同权重。
        *   **Numerical Claim Verification Tool (数值声明验证工具)**：使用Python代码解释器对文本中的数值声明进行精确计算和逻辑验证，防止数字误导。
    *   **记忆机制**：维护一个持久的工作记忆（Working Memory），存储所有中间结果，供不同工具共享上下文。

3.  **证据综合与结论生成模块 (Evidence Synthesis and Conclusion Generation)**
    *   **职责**：作为“最终裁判”，汇总所有证据生成最终报告。
    *   **技术/动作**：
        *   **证据整合**：根据可信度权重筛选冲突证据，保留高可信度信息。
        *   **推理链构建**：显式地将证据与结论链接起来，构建逻辑链条。
        *   **报告生成**：输出包含最终判定（真/假/部分正确）、置信度、完整引用和局限性讨论的详细报告，而非简单的二元标签。









## Improving Factuality and Reasoning in Language Models through Multiagent Debate (通过多智能体辩论提高语言模型的事实性和推理能力)

该论文解决了LLM在生成任务中事实性和推理准确性的核心痛点：
1. **单一模型的幻觉与推理缺陷**：
   - 现有的LLM经常自信地产生事实错误的幻觉，或在复杂推理链中出现逻辑跳跃。
   - 传统的单模型优化方法（如思维链CoT、自我反思）受限于模型自身的内部偏见，缺乏外部视角的纠错能力。
   - **解决方案**：提出一种“多智能体辩论”框架，受“心智社会（Society of Mind）”启发，让多个LLM实例针对同一问题提出不同见解，并通过多轮辩论相互纠错。
2. **事实一致性与共识缺失**：
   - 对于不确定的事实，不同模型实例可能生成不一致的答案。
   - **解决方案**：通过多轮互相批评和论证，迫使模型放弃不确定的错误信息，最终收敛到一个更准确的共同答案（Consensus）。

#### 具体Agent分工：输入问题 -> 独立生成 -> 多轮辩论循环（阅读他人观点 -> 更新自身观点） -> 输出共识答案

1. **辩论智能体 (Debating Agents / LLM Instances)**
   - **职责**：既是“解答者”也是“批评者”。它们负责生成初始答案，审查其他智能体的回答，并根据反馈修正自己的推理。
   - **技术/动作**：
     - **独立提案**：首先在零样本情况下独立生成问题的候选答案。
     - **交叉互评**：在每一轮辩论中，接收所有其他智能体的回答作为上下文输入。
     - **自我修正与更新**：基于其他智能体的逻辑和证据，使用特定的提示词（如“参考其他代理的方案作为额外建议，给出更新后的回答”）来优化自己的答案。这一过程重复多轮，直到达成共识或达到预设轮次。

2. **摘要智能体 (Summarization Agent，可选)**
   - **职责**：解决当参与辩论的智能体数量过多时，上下文长度超限的问题。
   - **技术/动作**：在将信息传递给下一轮辩论前，先将所有其他智能体的冗长回答进行摘要和提炼，再作为上下文输入喂给辩论智能体。









## Claim Veracity Assessment for Explainable Fake News Detection (基于声明真实性评估的可解释性虚假新闻检测)

该论文主要解决了虚假新闻检测中的两个核心痛点：

1.  **缺乏证据支持与可信度**：
    *   现有的检测方法大多依赖神经网络对文本特征进行二元分类（真/假），或者仅利用LLM生成缺乏事实依据的解释。
    *   这些方法往往不引用外部事实证据，也不提供证据来源，导致预测结果缺乏可信度，容易出现“幻觉”现象（即模型编造虚假解释）。
    *   **解决方案**：提出一种结合检索增强生成（RAG）和多模态（LLM+PLM）的框架，通过检索外部权威证据（如NIH, CDC等）来支撑真实性预测，并生成基于证据的解释。
2.  **解释生成的质量与准确性**：
    *   单纯依靠常识或文本风格分析的解释可能偏离主题或做出错误判断。
    *   **解决方案**：设计了一个“顾问模型（Advisor Model）”和“解释选择器（Explanation Chooser）”，能够从多种生成的解释（基于证据、基于常识、基于文本描述）中智能选择最合理、最符合事实的解释。

#### 工作流：输入声明 -> 多路解释生成 -> 顾问评估与选择 -> 输出最终预测与解释

1.  **无证据解释生成模块 (Module A)**
    *   **职责**：在不依赖外部数据库的情况下，利用LLM的内在知识进行快速初步判断。
    *   **技术/动作**：
        *   **基于常识的生成 (Commonsense Generation)**：利用LLM（如LLAMA3）根据逻辑推理评估声明的合理性。
        *   **基于文本描述的生成 (Textual Desc. Explanation)**：利用LLM分析声明的写作风格（如是否夸张、情绪化）来辅助判断。

2.  **有证据解释生成模块 (Module B - RAG)**
    *   **职责**：通过检索外部权威数据库，提供基于事实的强力证据支持。
    *   **技术/动作**：
        *   **查询生成 (Query Generation)**：将原始声明改写为更通用的查询问题，提高检索命中率。
        *   **证据搜索 (Evidence Search)**：在FAISS向量数据库中检索相关文章（来源包括NIH, CDC, Politifact等）。
        *   **解释生成 (Explanation Generation)**：将检索到的证据和原始声明一起喂给LLM，生成包含引用来源的真实性判断和详细解释。

3.  **解释选择器模块 (Module C - Explanation Chooser)**
    *   **职责**：作为系统的“大脑”，负责综合各方意见，选出最佳的最终解释。
    *   **技术/动作**：
        *   **顾问模型 (Advisor Model)**：一个微调过的RoBERTa模型，接收原始声明及上述三种生成的解释作为输入，预测一个“建议真实性标签（Advised Veracity）”。
        *   **选择逻辑**：对比各模块的预测结果与顾问模型的建议。优先选择与顾问建议一致且基于证据的解释；若不一致，则考虑其他类型的解释。最终输出不仅包含真/假标签，还包含一段高质量的、引用了证据的文本解释。











## Stance Detection with Collaborative Role-Infused LLM-Based Agents (基于角色注入协作型LLM智能体的立场检测)

该论文主要解决了立场检测任务中的两个核心痛点：

1.  **立场检测需要多方面的知识**：
    *   网络和社交媒体上的文本往往包含特定领域的术语、文化引用、俚语等，LLM可能无法直接理解这些元素的深层含义。
    *   **解决方案**：设计“多维文本分析阶段”，引入三个不同角色的智能体（语言专家、领域专家、社交媒体老手），分别从语言学、特定领域知识和社交媒体表达习惯三个维度对文本进行深度剖析。
2.  **立场检测需要高级推理能力**：
    *   作者往往不直接表达立场，而是通过对他人的态度、反讽或隐晦的逻辑链条来暗示。现有的LLM直接应用时，推理能力可能不足以捕捉这些细微差别。
    *   **解决方案**：引入“推理增强辩论阶段”，让不同的智能体分别扮演不同立场的辩护者（支持、反对、中立），基于前一阶段的分析结果寻找证据并进行辩论，迫使模型建立文本特征与立场之间的逻辑联系。

![PixPin_2026-01-21_21-11-34](./pics/PixPin_2026-01-21_21-11-34.png)

#### 工作流：输入文本 -> 多维文本分析 -> 推理增强辩论 -> 立场结论 -> 输出立场

1.  **多维文本分析智能体 (Multidimensional Text Analysis Agents)**
    *   **职责**：作为系统的“眼睛”和“大脑皮层”，负责解构文本，提取深层含义。
    *   **技术/动作**：
        *   **语言专家 (Linguistic Expert)**：分析语法结构、时态、修辞手法和措辞选择。
        *   **领域专家 (Domain Specialist)**：识别文本中的关键人物、事件、组织、政党和宗教信仰，并解释它们与目标立场的关联。
        *   **社交媒体老手 (Social Media Veteran)**：解读标签（Hashtags）、网络俚语、表情符号和情感基调。

2.  **推理增强辩论智能体 (Reasoning-Enhanced Debating Agents)**
    *   **职责**：作为系统的“辩手”，通过对抗性思考挖掘潜在逻辑。
    *   **技术/动作**：
        *   针对每个可能的立场（支持、反对、中立），分别指派一个智能体。
        *   每个辩手智能体利用第一阶段的分析结果作为证据，构建逻辑链条，为自己代表的立场进行辩护。
        *   通过这种“自我辩论”或“多智能体辩论”的形式，激发LLM的深度推理能力。

3.  **立场结论智能体 (Stance Conclusion Agent)**
    *   **职责**：作为系统的“法官”，做出最终裁决。
    *   **技术/动作**：
        *   接收原始文本以及上一阶段所有辩手智能体的辩词。
        *   综合评估各方论据的合理性和证据的强弱，最终判定文本的真实立场（Favor/Against/Neutral），并能提供简短的解释。





## Large Language Model Agent for Fake News Detection (基于大语言模型智能体的假新闻检测)

该论文主要解决了假新闻检测中的以下核心痛点：

1.  **现有方法的局限性**：
    *   **人工核查**：速度慢、成本高、无法应对海量信息。
    *   **传统监督学习模型**：依赖大量标注数据，数据获取困难且难以应对新领域。
    *   **现有LLM方法**：虽然无需训练，但往往是“非代理式（non-agentic）”的，即一次性给出答案，缺乏系统的验证过程，容易产生幻觉，且缺乏可解释性。
2.  **假新闻的多面性**：假新闻检测需要综合考虑真实性、作者意图、写作风格等多个维度，单一视角的检测往往不够准确。

**解决方案**：提出了 **FactAgent**，一种基于 LLM 的智能体方法。它通过模拟人类专家的事实核查行为，将复杂的验证任务分解为结构化的工作流。在这个工作流中，LLM 结合内部知识和外部工具（如搜索引擎），逐步收集证据并进行推理，最终做出判断。

![PixPin_2026-01-21_21-12-33](./pics/PixPin_2026-01-21_21-12-33.png)

#### 工作流：输入新闻 -> 工具调用与证据收集 -> 综合推理与决策 -> 输出结果

1.  **工具集 (Tools)**
    *   **职责**：从不同维度对新闻进行分析和取证。
    *   **具体工具与动作**：
        *   **Phrase_tool (措辞工具)**：利用LLM内部知识，检查新闻是否使用了煽动性、情绪化或夸张的语言（这类语言常用于假新闻）。
        *   **Language_tool (语言工具)**：利用LLM内部知识，检查语法错误、拼写错误或过度使用大写字母等低级错误。
        *   **Commonsense_tool (常识工具)**：利用LLM内部常识，判断新闻内容是否违背常识或更像是八卦而非事实报道。
        *   **Standing_tool (立场工具)**：专门针对政治新闻，利用LLM判断新闻是否带有明显的政治偏见，试图操纵舆论而非陈述客观事实。（注：若新闻非政治类，此工具会被跳过）。
        *   **Search_tool (搜索工具)**：调用外部 API (SerpApi) 搜索相关报道，寻找相互冲突的信息或缺乏证据支持的迹象，以解决LLM幻觉问题。
        *   **URL_tool (链接工具)**：结合LLM知识和外部数据库，评估新闻来源域名的可信度。

2.  **结构化专家工作流 (Structured Expert Workflow)**
    *   **职责**：协调各个工具的执行顺序和逻辑。
    *   **技术/动作**：
        *   首先判断新闻是否涉及政治。
        *   依次调用上述工具，收集每个工具的分析结果（Observations）。
        *   将复杂的验证任务分解为多个简单的子步骤，确保每个方面都得到检查。

3.  **决策与解释模块 (Decision Making & Explanation)**
    *   **职责**：综合所有证据，做出最终判断并提供解释。
    *   **技术/动作**：
        *   将所有工具收集到的观察结果汇总。
        *   LLM 对照一份“专家检查清单（Expert Checklist）”，逐一评估各项指标。
        *   最终生成真/假判断，并提供详细的推理过程，增强结果的可解释性。





## FACT-AUDIT: An Adaptive Multi-Agent Framework for Dynamic Fact-Checking Evaluation of Large Language Models (FACT-AUDIT：用于大语言模型动态事实验证评估的自适应多智能体框架)

该论文主要解决了LLM事实验证能力评估中的核心痛点：
1.  **静态数据集的局限性**：
    -   现有的自动评估方法依赖静态数据集，无法动态揭示LLM在事实验证中的细微局限性，容易出现数据泄漏和排行榜刷榜问题。
    -   **解决方案**：基于**重要性采样（Importance Sampling）**原理，设计一个自适应框架，根据模型表现动态生成和更新测试用例，专注于挖掘模型的弱点。
2.  **评估维度的单一性**：
    -   传统评估仅关注分类准确率（True/False），忽视了对模型生成理由（Justification）的评估，无法全面衡量模型的可信度。
    -   **解决方案**：引入双重评估机制，不仅评估判决结果，还通过LLM-as-a-Judge评估生成解释的质量，提出IMR（洞察掌握率）和JFR（理由缺陷率）等新指标。
3.  **复杂场景的覆盖不足**：
    -   现实世界中的虚假信息（如复杂主张、假新闻、谣言）具有开放性和动态性，现有方法难以大规模覆盖。
    -   **解决方案**：构建多智能体协作系统，模拟多种事实验证场景（Complex Claim, Fake News, Social Rumor）和测试模式（[claim], [evidence], [wisdom of crowds]），实现全面覆盖。

<img src="./pics/PixPin_2026-01-21_21-13-34.png" alt="PixPin_2026-01-21_21-13-34" style="zoom:50%;" />

![PixPin_2026-01-21_21-13-47](./pics/PixPin_2026-01-21_21-13-47.png)

#### 工作流：原型仿真 -> 事实核查与理由生成 -> 自适应更新 -> 循环迭代

1.  **原型仿真阶段 (Prototype Emulation)**
    -   **Appraiser Agent (评估者)**
        -   **职责**：制定和更新事实验证场景的分类体系（Taxonomy）。
        -   **技术/动作**：初始化分类体系（包含复杂主张、假新闻、社会谣言三大类及子类），并根据后续评估反馈，发现模型薄弱点，自适应地添加新的测试场景。
    -   **Inquirer Agent (询问者)**
        -   **职责**：根据分类体系生成具体的测试用例（Test Cases）。
        -   **技术/动作**：生成包含关键点、源主张、辅助信息（如维基百科证据或模拟社交评论）和测试模式的结构化数据。
    -   **Quality Inspector Agent (质量检查员)**
        -   **职责**：作为守门员，确保生成的测试用例质量。
        -   **技术/动作**：使用外部工具（如Wikipedia API）验证辅助信息的可靠性，并检查测试用例是否符合多样性、非多模态等要求。

2.  **事实核查与理由生成阶段 (Fact Verification with Justification)**
    -   **Target LLM (被测模型)**
        -   **职责**：作为被评估对象，执行事实验证任务。
        -   **技术/动作**：输入测试用例，输出判决结果（Factual/Non-Factual/NEI）及对应的理由（Justification）。
    -   **Evaluator Agent (评价者)**
        -   **职责**：评估被测模型的表现。
        -   **技术/动作**：采用LLM-as-a-Judge模式，对比被测模型的输出与参考答案，从判决准确性和理由质量两个维度进行打分（1-10分）和点评。特别地，若理由质量差，即使判决正确也会给予低分。

3.  **自适应更新与迭代探测阶段 (Adaptive Updating & Iterative Probing)**
    -   **Prober Agent (探测者)**
        -   **职责**：在特定场景下深入挖掘模型的弱点。
        -   **技术/动作**：基于历史评估记录（Memory Pool），通过少样本学习生成更具挑战性、更多样化的新测试用例，进行迭代式攻击探测。
    -   **反馈循环**：评估结果反馈给Appraiser，触发分类体系的更新（如细化某个模型表现差的子类别），从而进入下一轮更具针对性的评估循环。

















## The Truth Becomes Clearer Through Debate! Multi-Agent Systems with Large Language Models Unmask Fake News (真理越辩越明！基于大语言模型的多智能体系统揭露假新闻)

该论文主要解决了假新闻检测中的以下核心痛点：

1.  **可解释性差**：
    *   传统的基于深度学习的检测模型（如BERT）虽然在文本分析上表现出色，但往往是“黑盒”，缺乏对判断依据的清晰解释，导致用户难以信任。
    *   **解决方案**：引入多智能体辩论机制（TruEDebate），模拟人类的辩论过程，通过支持方和反对方的交锋，不仅得出结论，还展示了详细的推理逻辑和论证过程，大大增强了可解释性。
2.  **推理能力未充分利用**：
    *   现有利用LLM的方法多是直接通过特定提示词生成结果，未能充分挖掘LLM在复杂认知和逻辑推理方面的潜力，容易导致结论过于简单或草率。
    *   **解决方案**：设计结构化的辩论流程（开篇陈词、交叉盘问、结案陈词），迫使LLM智能体从不同角度深入剖析新闻内容，激发其深层推理能力。

<img src="./pics/PixPin_2026-01-21_21-15-19.png" alt="PixPin_2026-01-21_21-15-19" style="zoom:50%;" />

![PixPin_2026-01-21_21-15-25](./pics/PixPin_2026-01-21_21-15-25.png)

#### 工作流：新闻输入 -> 辩论流程（开篇 -> 盘问 -> 总结） -> 综合分析 -> 最终判定

1.  **辩论流智能体 (DebateFlow Agents)**
    *   **职责**：模拟辩论双方，对新闻的真实性进行全面探讨。
    *   **技术/动作**：
        *   **分组**：分为正方（Proponents，支持新闻为真）和反方（Opponents，认为新闻为假）。
        *   **辩论阶段**：
            *   **开篇陈词 (Opening Statement)**：双方各抒己见，提出初始论点。
            *   **交叉盘问与反驳 (Cross-examination and Rebuttal)**：双方针对对方的观点进行提问和反驳，挖掘逻辑漏洞。
            *   **结案陈词 (Closing Statement)**：总结己方观点，重申核心论据。
        *   所有交互记录在“辩论日志 (Debate Log)”中。

2.  **洞察流智能体 (InsightFlow Agents)**
    *   **职责**：基于辩论内容进行总结和最终裁决。
    *   **子智能体 1：合成智能体 (Synthesis Agent)**
        *   **职责**：作为辩论的记录员和初步分析员。
        *   **技术/动作**：分析辩论日志，生成一份包含双方关键论点、反驳点及整体辩论态势的“辩论报告 (Debate Report)”。
    *   **子智能体 2：分析智能体 (Analysis Agent)**
        *   **职责**：作为最终的法官，做出真假判定。
        *   **技术/动作**：
            *   **角色感知编码器 (Role-aware Encoder)**：结合辩论角色的身份（正方/反方）和其发言内容，生成带有角色信息的嵌入向量。
            *   **辩论图 (Debate Graph)**：构建图结构模型，节点为辩论交互，边为逻辑关系，使用图注意力网络 (GAT) 捕捉辩论的动态流向。
            *   **交互注意力机制 (Interactive Attention)**：结合原始新闻内容和辩论图的特征，计算最终的真假概率，并输出判定结果。







## The Stepwise Deception: Simulating the Evolution from True News to Fake News with LLM Agents (步步欺骗：利用 LLM 智能体模拟从真新闻到假新闻的演变)

该论文主要解决了假新闻研究中的两个核心痛点：

1.  **缺乏对演变过程的关注**：
    *   以往研究通常假设假新闻一出现就是假的，忽略了真新闻如何在传播过程中逐步被扭曲、误解，最终演变为假新闻的过程（即“部分演变的假新闻”）。
    *   **解决方案**：提出了 **FUSE**（假新闻演变模拟框架），利用 LLM 智能体模拟真实的社交网络互动，重现新闻从真到假的渐进式演变。
2.  **缺乏基于语言的量化评估标准**：
    *   现有的评估多基于传播结构或简单的文本相似度，缺乏多维度、细粒度的语言学评估指标来衡量新闻内容的失真程度。
    *   **解决方案**：开发了 **FUSE-EVAL**，一个多维评估框架，从情感、新信息引入、确定性、风格、时间性和观点偏差六个维度量化新闻的演变程度。

#### 工作流：初始化 -> 每日互动与演变 -> 评估与干预

1.  **传播角色感知智能体 (Propagation Role-Aware Agents, PRA)**
    *   **职责**：模拟社交网络中的个体，根据自身角色和属性处理和传播新闻。
    *   **技术/动作**：
        *   **角色分配**：每个智能体被赋予特定角色：传播者（Spreader，夸大信息）、评论者（Commentator，添加主观解读）、验证者（Verifier，核查事实）、旁观者（Bystander，被动观察）。
        *   **记忆机制**：拥有分层记忆系统（短期记忆存储近期互动，长期记忆积累知识），通过反思机制更新信念。
        *   **决策过程**：基于角色、个人特质（大五人格）和记忆，利用 LLM 生成新的新闻内容并在网络中传播。

2.  **新闻演变模拟器 (News Evolution Simulator, NES)**
    *   **职责**：提供模拟环境，管理社交网络结构和时间推进。
    *   **技术/动作**：
        *   **网络构建**：生成高聚类、无标度或随机网络，模拟真实的社交连接。
        *   **每日推进**：模拟每日的智能体交互，记录新闻内容的变化。
        *   **干预机制**：监控新闻偏差，当偏差超过阈值时，引入“官方代理”发布权威信息进行干预。

3.  **FUSE-EVAL 评估模块**
    *   **职责**：量化新闻演变的程度。
    *   **技术/动作**：利用 GPT-4o-mini 从 6 个维度（情感偏移、新信息引入、确定性偏移、风格偏移、时间偏移、观点偏差）对演变后的新闻与原始新闻进行打分（1-10分），计算总偏差值。









## MIND: A Multi-agent Framework for Zero-shot Harmful Meme Detection (MIND：一种用于零样本有害模因检测的多智能体框架)

该论文主要解决了有害模因检测中的以下核心痛点：

1.  **数据标注滞后与演化快**：
    *   模因在社交媒体上通过修改图片或文字迅速演变（如替换政治人物、修改讽刺语），传统的监督学习模型依赖大量标注数据，难以应对新出现的模因变体。
    *   **解决方案**：提出一种**零样本（Zero-shot）**框架，不需要带标签的训练数据，而是利用未标注的参考集（Reference Set）作为上下文来源。

2.  **多模态隐喻理解难**：
    *   有害模因的危害性往往隐藏在图文结合的隐喻或反讽中，直接输入大模型（LMM）容易导致推理不准确或产生幻觉。
    *   **解决方案**：通过**双向洞察推导（Bi-directional Insight Derivation）**和**多智能体辩论（Multi-agent Debate）**机制，模拟人类通过对比类似案例来理解新事物的认知过程，从而提取深层语义。

#### 具体Agent分工与工作流：检索 -> 洞察推导 -> 辩论推理 -> 最终判决

工作流旨在模仿人类的协作分析过程：先找类似的例子，总结规律，然后应用规律进行判断，若有分歧则进行仲裁。

1.  **相似样本检索模块 (Similar Sample Retrieval, SSR)**
    *   **职责**：为目标模因寻找上下文参照物。
    *   **动作**：虽然不是显式的Agent，但作为前置步骤，使用CLIP模型计算多模态嵌入（Embedding），从未标注的参考集中检索出Top-K个最相似的模因。这些相似模因（通常是同一模因模板的不同变体）将作为后续Agent的知识源。

2.  **洞察推导智能体 (Insight Deriving Agent, $LMM_{deriving}$)**
    *   **职责**：从相似模因序列中提取判断有害性的“规则”或“洞察”。
    *   **技术/动作**：
        *   **双向推导**：为了避免顺序偏差，该Agent执行两次推导：
            *   **正向推导 (Forward)**：从相似模因1到K顺序分析，逐步累积“什么是这类模因的有害点”的洞察。
            *   **反向推导 (Backward)**：从相似模因K到1逆序分析，补充被忽略的细节。
        *   **输出**：生成两套洞察集合（正向洞察集 $I_{fwd}$ 和 反向洞察集 $I_{back}$）。

3.  **辩论智能体 (Debater Agents, $LMM_{debater}$)**
    *   **职责**：基于不同的洞察来源，对目标模因进行独立判断。
    *   **技术/动作**：
        *   **正向辩手**：利用“正向洞察集” + “目标模因”，进行推理并给出判断（有害/无害）。
        *   **反向辩手**：利用“反向洞察集” + “目标模因”，进行推理并给出判断。
        *   **目的**：通过不同的推理路径（思维链）来增加决策的鲁棒性。

4.  **法官智能体 (Judge Agent, $LMM_{judge}$)**
    *   **职责**：解决冲突，做出最终决策。
    *   **技术/动作**：
        *   **共识机制**：如果正向辩手和反向辩手的判断一致（例如都认为有害），直接输出该结果。
        *   **仲裁机制**：如果两名辩手意见不一致，法官Agent会审查双方的推理过程（Reasoning）和论据，进行权衡，最终输出定论。







## PREDICT: Multi-Agent-based Debate Simulation for Generalized Hate Speech Detection (PREDICT：基于多智能体辩论模拟的广义仇恨言论检测)

该论文主要解决了由于不同数据集标注标准不一致导致的仇恨言论检测模型泛化能力差的问题：

1.  **痛点一：数据集标注标准差异导致模型泛化困难**
    *   不同公开数据集对“仇恨言论”的定义侧重点不同（如关注情感、特定目标群体或社会历史背景），导致在某一数据集上训练的模型在其他数据集上表现不佳。
    *   **解决方案**：提出了PREDICT框架，利用多智能体（Multi-Agent）模拟不同数据集的标注视角。通过将不同数据集的标注标准和相似上下文转化为独立的“视角”，让多个智能体分别依据这些视角对同一文本进行推理。

2.  **痛点二：单一视角或简单投票无法有效处理复杂争议**
    *   简单的多数投票机制忽略了少数派意见和标准间的细微差别，且单一模型容易过拟合特定数据的偏见。
    *   **解决方案**：引入辩论机制（DICT阶段）。将不同视角产生的理由汇聚成“仇恨”和“非仇恨”两组参考信息，强制辩论代理利用这些不一致的参考信息进行辩论，最终由法官代理综合双方论点做出平衡的决策。

#### 工作流：输入文本 -> PRE阶段(多视角推理) -> 理由聚合 -> DICT阶段(辩论模拟) -> 法官裁决

1.  **视角代理 (Perspective Agents)**
    *   **职责**：模拟特定数据集（如K-HATERS, K-MHaS等）的标注者，针对输入文本生成“仇恨”或“非仇恨”的立场（Stance）及理由（Reason）。
    *   **技术/动作**：
        *   **独立视角构建**：基于特定数据集的标注指南（Labeling Criteria）和通过余弦相似度检索到的相似上下文（Similar Contexts/Few-shot examples）构建Prompt。
        *   **推理生成**：每个代理独立运行，输出其立场和理由。

2.  **参考信息聚合模块 (Reference Aggregator)**
    *   **职责**：将PRE阶段所有视角代理生成的理由按立场分类，形成“仇恨参考库（Hate Reference）”和“非仇恨参考库（Non-hate Reference）”。
    *   **技术/动作**：简单的逻辑分类，若某代理判定为仇恨，其理由加入仇恨库；反之加入非仇恨库。这为后续辩论提供了依据（References）。

3.  **辩论主持人 (Moderator Agent)**
    *   **职责**：引导辩论流程，发布辩题，并要求辩手进行陈述或反驳。
    *   **技术/动作**：基于规则的Prompt模板，控制辩论的轮次（共两轮：第一轮陈述观点，第二轮反驳或同意）。

4.  **辩手代理 (Debater Agents - Hate & Non-hate)**
    *   **职责**：分别代表“仇恨”方和“非仇恨”方进行辩论。
    *   **技术/动作**：
        *   **基于参考的论证**：在第一轮中，强制引用各自阵营的参考库（References）来支持其观点，即使该观点可能与模型内部知识冲突。
        *   **反驳与修正**：在第二轮中，根据对手的论点选择反驳或承认（Agree/Rebut），并允许修正初始立场。

5.  **法官代理 (Judge Agent)**
    *   **职责**：根据辩论历史记录，做出最终的标签分类（仇恨/非仇恨）并提供平衡的理由。
    *   **技术/动作**：接收完整的辩论历史（包含双方的论点交互），通过Prompt指示其综合考虑少数派意见和双方论据的合理性，输出最终决策（JSON格式）。







## Unleashing the Potential of Large Language Models as Prompt Optimizers: Analogical Analysis with Gradient-based Model Optimizers (释放大型语言模型作为提示优化器的潜力：与基于梯度的模型优化器的类比分析)

该论文解决了大型语言模型（LLM）作为提示优化器（Prompt Optimizer）时设计缺乏理论指导和效率低下的痛点：

1.  **痛点一：Meta-Prompt设计缺乏理论指导与系统性**
    *   现有的自动提示优化方法在设计Meta-Prompt（指导LLM如何优化提示的提示）时，往往依赖人工试错或启发式算法，缺乏原则性的指导方针。
    *   **解决方案**：提出了一种类比分析法，将LLM提示优化与成熟的“基于梯度的模型优化”（如SGD）进行对应。将模型参数学习中的“更新方向（Update Direction）”和“更新方法（Update Method）”两个关键因素映射到提示优化中，以此指导Meta-Prompt的设计。

2.  **痛点二：优化过程不稳定且难以利用历史信息**
    *   简单的优化方法（如直接利用反思或随机搜索）往往忽略了优化过程中的历史轨迹，或者更新步长难以控制，导致震荡或陷入次优解。
    *   **解决方案**：开发了GPO（Gradient-inspired Prompt Optimizer）。它利用“相关性轨迹（Relevance-based Trajectory）”模拟动量（Momentum）来确定更新方向，并利用“余弦衰减策略（Cosine-based Decay）”控制编辑距离来模拟学习率（Learning Rate），从而实现更稳定高效的迭代。

![PixPin_2026-01-21_21-30-17](./pics/PixPin_2026-01-21_21-30-17.png)

#### 工作流：初始提示 -> 轨迹检索(动量模拟) -> Meta-Prompt构建 -> 提示生成(生成式细化) -> 任务评估 -> 迭代循环

1.  **轨迹检索器 (Trajectory Retriever)**
    *   **职责**：确定优化的“更新方向”，模拟梯度下降中的动量（Momentum）。
    *   **技术/动作**：在每一步优化中，不是简单使用上一轮结果，而是从历史优化轨迹中检索与当前提示语义最相似的$k$个中间提示及其得分（Relevance-based strategy）。这些历史提示按相似度排序，作为上下文提供给优化器，帮助模型捕捉全局优化方向。

2.  **提示优化代理 (Prompt Optimizer Agent - GPO)**
    *   **职责**：基于检索到的轨迹和当前状态，生成新的、性能更好的任务提示。
    *   **技术/动作**：
        *   **生成式细化 (Generation-based Refinement)**：模拟“下降”步骤。不要求模型直接编辑原提示，而是让模型根据通过Few-shot提供的轨迹演示，直接生成全新的提示。
        *   **余弦衰减约束 (Cosine-based Decay Constraint)**：模拟“学习率调度”。通过在Meta-Prompt中限制允许修改的词数或变化程度，随着迭代次数增加，逐渐减小提示的变动范围（从探索转向利用），防止过度优化。

3.  **任务评估代理 (Task Evaluator / Task Model)**
    *   **职责**：验证优化后的提示在具体任务上的表现。
    *   **技术/动作**：将GPO生成的候选提示应用于目标任务模型（如Llama-2或GPT-4），在训练集/验证集上执行并计算准确率（如Exact Match）。评估结果（Score）会反馈给系统，用于更新优化轨迹库，供下一轮检索使用。







## LLM-GAN: Construct Generative Adversarial Network Through Large Language Models For Explainable Fake News Detection (LLM-GAN：通过大型语言模型构建生成对抗网络以实现可解释的假新闻检测)

该论文解决了LLM在可解释性假新闻检测中面临的两个核心痛点：

1.  **痛点一：假新闻的高欺骗性导致LLM难以识别**
    *   假新闻通常编写得合理且连贯，容易误导LLM产生“证实偏差”（即倾向于认为新闻是真实的），因为LLM缺乏对复杂造假过程的理解。
    *   **解决方案**：提出了**对抗性提示（Inter-adversary Prompting）**机制。模仿GAN的结构，引入一个LLM生成器专门制造高欺骗性的假新闻来“攻击”检测器，迫使检测器在对抗中学习造假者的手段和逻辑，从而提升识别能力。
2.  **痛点二：LLM生成的解释质量参差不齐且依赖人工修正**
    *   直接使用LLM进行检测时，往往会生成包含逻辑错误的解释，修正这些错误通常需要大量昂贵的人工成本。
    *   **解决方案**：提出了**自反思提示（Self-reflection Prompting）**机制。引入一个LLM反思器，自动分析检测器的错误预测并生成反馈，使检测器能够自主从过去的错误中学习并更新策略，实现全自动化的流程。

![PixPin_2026-01-21_21-31-23](./pics/PixPin_2026-01-21_21-31-23.png)

![PixPin_2026-01-21_21-31-34](./pics/PixPin_2026-01-21_21-31-34.png)

#### 工作流：真实新闻采集 -> 对抗性提示阶段(生成器与检测器博弈) -> 自反思提示阶段(反思器修正) -> 最终预测与解释

1.  **Generator Agent (生成器)**
    *   **职责**：基于真实的种子新闻，利用特定的伪造策略生成极具欺骗性的假新闻，目标是愚弄检测器。
    *   **技术/动作**：接收真实新闻$x$和伪造策略$S_G$作为输入生成假新闻$x'$。如果生成的假新闻被检测器成功识破，生成器会利用检测器提供的“检测解释”来升级自己的伪造策略，以在下一轮生成更难辨别的样本。
2.  **Detector Agent (检测器)**
    *   **职责**：负责区分新闻的真伪，并输出详细的判别理由（Explanation）。
    *   **技术/动作**：
        *   **对抗阶段**：判断生成器的输出。如果判断错误（被愚弄），它会利用生成器提供的“伪造解释”来更新自身的检测策略$S_D$；如果判断正确，则保持优势。
        *   **反思阶段**：接收反思器提供的反馈，修正其对历史错误样本的理解。
3.  **Reflector Agent (反思器)**
    *   **职责**：作为自动化纠错机制，专门监控检测器在训练数据上的表现，针对错误案例提供指导。
    *   **技术/动作**：当检测器对某个样本判断错误时，反思器会生成“反思反馈（Reflection Feedback）”，明确指出检测解释中的逻辑漏洞并提供正确证据。这些反馈被直接用于更新检测器的Prompt策略，使其实现自我修正。







## ContextAware: A Multi-Agent Framework for Detecting Harmful Image-Based Comments on Social Media (ContextAware：一种用于检测社交媒体上有害图片评论的多智能体框架)

该论文旨在解决社交媒体中基于图片的隐性污名化评论难以检测的痛点：

1.  **痛点一：隐性污名化内容具有高度隐蔽性和多模态复杂性**
    *   传统的基于关键词或视觉特征的方法难以捕捉包含讽刺、隐喻或特定文化暗示的隐性有害内容，且文本与图像之间常存在语义未对齐的问题。
    *   **解决方案**：提出了ContextAware框架，利用多智能体（Multi-Agent）架构，整合大语言模型（GPT-4o）和视觉语言模型（CLIP），协同分析图像、文本语境及社交互动线索。
2.  **痛点二：现有方法忽视了关键的上下文与社交信号**
    *   现有检测模型往往忽略了视频标题、标签（Hashtags）以及评论的“点赞数”等社交元数据，而这些信息对于理解图片评论的真实意图至关重要。
    *   **解决方案**：框架专门设计了处理视频元数据和社交信号的模块，优先分析高点赞（高曝光）内容，并结合评论区整体情感氛围来辅助判断图片的有害性。
3.  **痛点三：缺乏针对特定文化背景下污名化模式的泛化能力**
    *   不同文化背景下的歧视表达方式（如特定模因Meme）差异巨大且演变迅速，通用模型难以覆盖。
    *   **解决方案**：引入了“基于案例的学习（Case-based Learning）”机制，从当前数据集中自动聚类并提炼出特定的污名化原则和模式，作为先验知识辅助最终决策。

#### 工作流：图像聚类去重 -> 文本语境提取 -> 高赞图片初筛 -> 案例原则提取 -> 综合决策

1.  **Image Embedding Database (图像嵌入数据库)**
    *   **职责**：对评论区的所有图片进行特征提取和聚类，识别近似重复的图片，避免重复计算。
    *   **技术/动作**：使用预训练的 **CLIP (ViT-B/32)** 模型计算图像嵌入，并采用 **DBSCAN** 算法聚类（参数eps=0.18, min_samples=2），将相似度极高的图片视为同一组进行处理。

2.  **Comment Text Processing Agent (评论文本处理智能体)**
    *   **职责**：理解评论区的整体舆论氛围、主要话题和情感倾向，作为辅助信号。
    *   **技术/动作**：选取点赞数最高的20条评论及随机5条评论，结合视频标题和标签，利用 **GPT-4o** (Temperature=0) 推断整体情感（积极/消极/中性）及潜在的讽刺或有害基调。

3.  **High-likes Image Analysis Agent (高赞图片分析智能体)**
    *   **职责**：利用社交信号（点赞数）筛选出最可能包含有害信息的高热度图片，进行初步过滤。
    *   **技术/动作**：提取每个视频下点赞数前10%的图片。结合文本代理提供的情感语境和视频元数据，使用 **GPT-4o** 进行初步检测，标记出“潜在有害”的图片。

4.  **Case-based Learning Agent (基于案例的学习智能体)**
    *   **职责**：对初筛出的潜在有害图片进行深度分析，提炼出隐性污名化的通用原则（知识库）。
    *   **技术/动作**：首先使用 **GPT-4o** (Temperature=0.5) 分析图片中的隐喻、刻板印象或种族讽刺并生成解释；然后利用 **SentenceTransformer (m3e-base)** 将解释向量化，通过 **DBSCAN** 聚类，总结出如“夸张的面部特征+特定种族的负面文本”等具体的污名化原则。

5.  **Comprehensive Analysis Agent (综合分析智能体)**
    *   **职责**：整合多方信息，对非高赞图片或存疑图片进行最终的二分类判定（有害/无害）。
    *   **技术/动作**：输入包括：文本代理的情感分析结果、案例学习代理提炼的危害原则、以及原始图片。使用 **GPT-4o** 进行最终推理，利用学到的原则解决文本与图像证据冲突的情况，输出最终分类结果。





## Who’s the Mole? Modeling and Detecting Intention-Hiding Malicious Agents in LLM-Based Multi-Agent Systems (谁是内鬼？LLM多智能体系统中隐藏意图恶意智能体的建模与检测)

该论文解决了LLM多智能体系统（LLM-MAS）中恶意智能体通过隐藏意图破坏协作且难以被现有机制检测的痛点：

1.  **痛点一：恶意行为具有高度隐蔽性，现有防御机制失效**
    *   现有的安全研究主要针对单智能体的显式攻击（如提示注入、越狱），而忽略了在多智能体协作中，恶意智能体可以通过表面上看似合作、符合角色设定的行为（如提出次优方案、巧妙转移话题、伪造共识或故意拖延）来破坏任务目标。这些“隐藏意图”的攻击能够轻易绕过基于输出过滤或规则一致性的传统检测机制。
    *   **解决方案**：论文首先定义并验证了四种隐藏意图攻击范式（次优固着、重构错位、虚假注入、执行延迟），随后提出了 **AgentXposed** 检测框架。该框架基于心理学理论，利用 **HEXACO人格模型**（特别是诚实-谦逊维度）来刻画智能体特征，并结合 **Reid审讯技术** 通过结构化提问来主动诱发并暴露恶意意图。

2.  **痛点二：缺乏针对多智能体复杂通信结构的安全评估与防御**
    *   多智能体系统存在中心化、去中心化和分层等多种通信结构，恶意行为在不同结构下的传播和破坏力不同，缺乏系统性的评估和有效的应对策略。
    *   **解决方案**：论文在三种通信结构和六个数据集上进行了系统性评估，证明了攻击的普遍有效性。基于检测结果，进一步提出了两种轻量级防御策略：**认知重定向引导（Cognitive Redirection Guide）**（通过语义引导修正智能体行为）和 **踢出（Kick）**（直接移除恶意智能体），以减轻攻击影响。

#### 具体Agent分工：输入（协作历史/任务） -> 步骤A（基线分析） -> 步骤B（异常监测） -> 步骤C（审讯验证） -> 输出（恶意判定）

1.  **基线画像构建模块 (Baseline Profiler)**
    *   **职责**：在任务初始阶段或基于早期交互，建立每个智能体的心理行为基线。
    *   **技术/动作**：不依赖显式的问卷调查，而是基于智能体的角色设定（Role Prompt）和早期对话历史，利用 LLM 推断并量化其 **HEXACO** 人格维度得分（如诚实-谦逊、尽责性等），形成初始心理画像。

2.  **偏差敏感监测模块 (Deviation-Sensitive Monitor)**
    *   **职责**：在多智能体协作过程中，实时监控智能体的行为变化，识别潜在的异常。
    *   **技术/动作**：持续跟踪智能体的发言和工具使用行为，计算当前交互轮次下的人格评分与初始基线的偏差值（$\Delta_{i,t}$）。如果某智能体的行为与基线严重偏离（超过阈值 $\tau$），即判定其存在“行为不一致”，并触发后续的验证阶段。

3.  **审讯验证模块 (Verification Interrogator)**
    *   **职责**：对被标记为可疑的智能体进行针对性的主动试探，以确认其是否存在恶意意图。
    *   **技术/动作**：
        *   **Reid技术应用**：基于 **Reid审讯技术** 的原则，生成递进式的、具有心理压力的质询问题（Progressively Designed Inquiries）。这些问题旨在寻找逻辑漏洞、施加认知压力或诱导解释。
        *   **动态更新与判定**：根据智能体对质询的回复，动态更新其 HEXACO 评分。系统通过比较智能体在面对质询时的反应与正常行为模式的差异，最终判定其是否为“内鬼”。





## MultiPhishGuard: An LLM-based Multi-Agent System for Phishing Email Detection (MultiPhishGuard：基于大语言模型的多智能体钓鱼邮件检测系统)

该论文解决了网络钓鱼邮件检测中存在的以下核心痛点：

1.  **痛点一：传统检测方法难以适应快速演变的攻击策略**
    *   基于规则、黑名单或静态机器学习的方法依赖于预定义特征和历史数据，面对域欺骗、URL混淆或上下文感知的社会工程学攻击时往往失效。
    *   **解决方案**：提出了一个基于LLM的多智能体框架，结合**对抗训练（Adversarial Training）**模块。引入一个对抗智能体生成微妙的钓鱼邮件变体（如利用同义词替换、句式重写等），通过不断的攻防博弈来增强检测系统的鲁棒性和泛化能力。

2.  **痛点二：单一模态分析存在局限性且缺乏可解释性**
    *   现有深度学习方法通常只关注邮件文本或URL等单一模态，容易遗漏恶意线索；且“黑盒”特性导致缺乏透明的推理过程，用户难以理解为何邮件被标记为钓鱼邮件。
    *   **解决方案**：设计了**多模态协同机制**，利用专门的智能体分别分析文本、URL和元数据。同时引入**解释简化智能体（Explanation Simplifier Agent）**，将各组件的技术性判断整合为用户易懂的自然语言解释。此外，使用**近端策略优化（PPO）**强化学习算法，根据邮件的具体特征动态调整各智能体的决策权重，而非使用固定权重。

![PixPin_2026-01-21_21-36-27](./pics/PixPin_2026-01-21_21-36-27.png)

#### 工作流：邮件输入 -> 多智能体并行分析 -> RL动态权重融合 -> 对抗性反馈优化 -> 解释生成 -> 输出结果

1.  **文本分析智能体 (Text Analysis Agent)**
    *   **职责**：专门分析邮件正文内容，寻找恶意意图的文本线索。
    *   **技术/动作**：基于LLM分析异常语言模式、钓鱼关键词（如紧迫感词汇）和语义上下文，输出判断结果、置信度及理由。

2.  **URL分析智能体 (URL Analysis Agent)**
    *   **职责**：提取并审查邮件中的所有链接。
    *   **技术/动作**：检查链接混淆、评估域名声誉、检测潜在的重定向攻击，仅关注URL特征而不处理文本或元数据。

3.  **元数据分析智能体 (Metadata Analysis Agent)**
    *   **职责**：检查邮件头信息以识别伪造迹象。
    *   **技术/动作**：分析发件人身份验证记录（SPF, DKIM, DMARC）、Reply-to字段及Header异常，识别域名欺骗等技术性攻击。

4.  **对抗智能体 (Adversarial Agent)**
    *   **职责**：生成具有挑战性的对抗样本，用于训练和增强检测系统。
    *   **技术/动作**：基于GPT-4o，利用同义词替换、句子重写、内容修改、同形字替换（Homoglyph Replacement）等策略，生成旨在绕过检测的钓鱼邮件变体以及模拟钓鱼特征的合法邮件变体。

5.  **解释简化智能体 (Explanation Simplifier Agent)**
    *   **职责**：为非专家用户提供清晰、可读的决策解释。
    *   **技术/动作**：汇总文本、URL和元数据智能体的技术性理由，去除专业术语，合成一个连贯、简洁且基于事实的自然语言解释，说明判决依据。

6.  **PPO权重优化模块 (PPO Ensemble Weight Optimizer)**
    *   **职责**：动态调整上述三个分析智能体（文本、URL、元数据）在最终决策中的权重。
    *   **技术/动作**：使用近端策略优化（Proximal Policy Optimization）强化学习算法，根据每封邮件的特征（如URL数量、关键词）实时计算各智能体的最佳权重组合，以提高检测准确率。

## LoCal: Logical and Causal Fact-Checking with LLM based Multi-Agents (LoCal：基于LLM多智能体的逻辑与因果事实核查)

该论文解决了复杂事实验证中逻辑一致性和因果推理错误的痛点：

1.  **痛点一：逻辑组合错误与缺乏可解释性**
    *   现有的将复杂声明分解为子任务的方法，在将子任务结果重新组合成最终结论时容易出现逻辑错误，且往往是“黑盒”操作，缺乏可解释性。
    *   **解决方案**：提出了LoCal框架，通过多智能体协作生成结构化的解决方案（子任务-答案对）。引入“逻辑评估智能体”，专门检查生成的推理过程在逻辑上是否等同于原始声明，并提供详细的评估过程描述以增强可解释性。

2.  **痛点二：因果推理错误与幻觉**
    *   由于证据不足或LLM的幻觉，推理过程可能产生因果错误，导致得出不准确或非唯一的最终标签。
    *   **解决方案**：引入“反事实评估智能体”，通过假设相反的真伪标签来检测推理链中是否存在矛盾。只有当反事实假设导致矛盾时，原推断才被认为是具有因果鲁棒性的。系统根据评估结果迭代修正推理过程中的错误。

![PixPin_2026-01-21_21-37-51](./pics/PixPin_2026-01-21_21-37-51.png)

#### 具体Agent分工：输入声明 -> 分解 -> 检索与推理 -> 评估与迭代 -> 输出真伪

1.  **Decomposing Agent (分解智能体)**
    *   **职责**：负责将复杂的输入声明分解为更简单的原子子任务。
    *   **技术/动作**：利用LLM的上下文学习能力，生成子任务序列（包括事实验证任务和问答任务）以及用于后续推导最终真伪的逻辑函数。

2.  **Reasoning Agents (推理智能体)**
    *   **职责**：根据检索到的证据回答分解后的子任务。
    *   **技术/动作**：分为两类：
        *   **Fact-Validating Agent (事实验证智能体)**：针对原子声明类型的子任务，结合证据进行比对分析，返回布尔值（True/False）。
        *   **Question-Answering Agent (问答智能体)**：针对问题类型的子任务，从证据中提取信息并生成直接答案。

3.  **Logically Evaluating Agent (逻辑评估智能体)**
    *   **职责**：从逻辑角度验证推理过程的正确性。
    *   **技术/动作**：接收推理过程的摘要描述（子任务+答案），判断基于此摘要得出的新真伪结论是否与根据原始声明逻辑推导出的结论一致。

4.  **Counterfactually Evaluating Agent (反事实评估智能体)**
    *   **职责**：从因果角度验证结论的稳健性。
    *   **技术/动作**：强制否定预测的真伪标签（即假设相反结论成立），检查这是否会与推理摘要产生冲突。如果没有冲突，说明原结论不可靠（Reject）；如果产生矛盾，说明原结论成立（Accept）。

5.  **Confidence Updater (置信度更新器)**
    *   **职责**：根据评估结果决定是输出最终结果还是重新迭代。
    *   **技术/动作**：统计两个评估智能体的接受情况。若两者都接受，则输出高置信度结果；否则，根据拒绝情况更新计数器，并触发下一轮的“分解-推理-评估”循环，直到达到最大迭代次数或满足条件。
